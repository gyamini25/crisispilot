# CrisisPilot

> **Autonomous AI Incident Commander** — real-time detection, multi-agent investigation, and remediation orchestration for production incidents.

When a production incident fires, CrisisPilot spins up a war-room of specialized AI agents that investigate in parallel, correlate telemetry with recent deploys, quantify business impact, synthesize a root cause, and propose remediation — all streaming live to a dashboard, at human-readable cadence.

## Hackathon track & required integrations

This project targets the **Dynatrace** track and uses all three required building blocks:

| Requirement | How CrisisPilot uses it |
|---|---|
| **Gemini** | Every agent's reasoning is generated by Gemini (`gemini-flash-lite-latest`), streamed token-by-token to the UI. |
| **Google Cloud Agent Builder** | The 5 agents are real **ADK (Agent Development Kit) `LlmAgent`s** executed through ADK's `Runner` (`backend/app/agents/adk_runtime.py`). |
| **Partner MCP — Dynatrace** | Agents query **Dynatrace Grail via the official Dynatrace MCP server** (`@dynatrace-oss/dynatrace-mcp-server`) over stdio — real `execute_dql` calls for metrics and deployment events (`backend/app/integrations/dynatrace_mcp.py`). |

All three degrade gracefully: if a key/credential is absent or rate-limited, agents fall back to scripted-but-realistic reasoning so the demo never breaks.

## Architecture

```
                    ┌─────────────────────────────────────────────┐
   Incident  ──▶    │  Orchestrator (sequences the agent war-room) │
   (sim / API /     └─────────────────────────────────────────────┘
    file upload)             │ parallel              │ then
                    ┌────────┼────────┐         ┌────┴─────┐
                    ▼        ▼        ▼         ▼          ▼
                Metrics  Deployment Business  RootCause   Comms
                 Agent     Agent    Impact     Agent      Agent
                    │        │        │
                    ▼        ▼        ▼
            Dynatrace MCP (execute_dql / deployments)   ← real partner MCP
                    │
            each agent = ADK LlmAgent (Agent Builder)   ← Gemini-backed
                    │
            pub/sub bus ──▶ WebSocket ──▶ Next.js dashboard (live)
```

- **Frontend:** Next.js 15 (App Router), Tailwind, Framer Motion, React Flow
- **Backend:** FastAPI + WebSockets, async event bus → WS broadcast
- **Persistence:** MongoDB Atlas (Motor) with an in-memory fallback adapter
- **AI:** ADK `LlmAgent`s on Gemini; Dynatrace MCP for live observability data

## Quick start

Two terminals. Copy `backend/.env.example` → `backend/.env` and fill in keys
(all optional — the app runs in a degraded "scripted" mode without them).

### 1. Backend

```bash
cd backend
./run.sh        # creates .venv, installs deps, boots FastAPI on :8000
```

`.env` keys:
- `GEMINI_API_KEY` — enables live Gemini + ADK agents
- `DT_ENVIRONMENT` (`https://<env-id>.apps.dynatrace.com`) + `DT_PLATFORM_TOKEN` — enables the Dynatrace MCP integration
- `MONGODB_URI` — real Atlas persistence (omit for in-memory)
- `CRISISPILOT_USE_ADK=true` — run agents through ADK (default on)

Health check: `GET /healthz` reports which integrations are live.

### 2. Frontend

```bash
cd frontend
npm install
npm run dev     # http://localhost:3000
```

Open **http://localhost:3000**, hit **Trigger incident**, and watch the agents investigate.

## Demo loop

1. Boot both services.
2. Hit **Trigger incident** (or `POST /api/incidents/trigger`).
3. Watch Metrics → Deployment → BusinessImpact investigate in parallel, then RootCause synthesize, then Comms propose remediation — streaming live.
4. The war-room shows real **Dynatrace MCP** tool calls (`dynatrace.query`, `dynatrace.deployments`) annotated with their source.
